6.1 Font Properties

Example :

h2 {
    font-weight: bold
    font-size: 25px
    font-family: sans-serif
    text-algin: center
}

Font size logic
  • px: pixel
  • pt: similar to pixel
  • em: 100% size of parent
  • rem: 100% size of root
What does (em) and (rem) do exactly?

Ex : (em)
Lets say {h1: 1rem}, then if we had a parent tag on it h1 would take 100% the size of the parent tag

<body>
    <h1> Lebron James </h1>
</body>

If body was 20px then h1 in this case would be 20px. If h1 was 2em and body 20px then we get (20 * 2) = 40px as h1 size in this case


Ex : (rem)

Looking back to our previous case where body is 20px but now {h1: 2rem}, h1 is not affected by body anymore only the highest level (root) tag can ffect it, in most cases that is just the html tag.

Try to use this as a base for creating text sizes for your site


Font weight. (lighter, bolder) are relative to parent font weight while we can normally use (normal, bold)

Font familes. Simply the type of font we want to use with backups if it fails to work. Can also import fonts from outside sources using <link>.

6.3 CSS Box Model - Margin, Padding and Border properties


Allow elements to take up more or less space, depending on how we set it up.

Ex : for some element
height: 300px, 100% (of page size)
width: 300px, 100% (of page size)

This below makes a style=(solid) color=(balck) type=(border) thickness=(10px) thick around the box. Given (border: 10px solid black) we can do.

 border-top: 0px (no more boarder at top)
 border-width: 0px, 10px, 20px, 30px (0 top, 10 right, 20 bottom, 30 left)
 border-width: 0px, 10px (0 top, 10 right, 0 bottom, 10 left)

So we can have further specifications for box perameters, as well as padding around text.

  padding: 20px (text inside now surrounded by at least 20 more px on all sides to push boarder out)

margin will create a buffer between boarders.

  margin: 10px (10px buffer outside of the border)
