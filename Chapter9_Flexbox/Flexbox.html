9.2 Flex Layout

Is our property going to go to the child (boxes in felxbox) or parent (The flexbox container). Here are some stuff to help formatting the flexbox.

order: #; add this into a box to change where in the flexbox an item is displayed. For example setting (order: 1;) on a <div> with nothing else ordered would move the item to the furthest right/bottom box
since its set with the highest value by default. All boxes are initially set with an order 0, higher values are further right/bottom. This is a child attribute


flex-wrap: nowarp; by default flex box has its wrap setting set to no wrap, causing elements to possibly be pushed of a page.
We can change this property to flex-wrap: warp; moving the items that used to have no space onto the next line. This is a parent attribute


justify-content: flex-start; Sets the distrabution of our items on the main-axis. The base setting is flex-start, if we did flex-end then all items would be pushed to the right/top wall
of the flexbox, if extra space was left. center just puts the item in center between the leftover space. space-between/around/evenly gaps out the items to fill the leftover space.
This is a parent attribute


align-items: flex-start, same attribute names as justify-content, only this time it sets for the distribution along the cross-axis. Would need to set a size for this axis (preferably ##vh "veiwport height").


We can combine these two forms of movement for items to set boxes to different spaces.


align-content: cetner; This only works if the flex-wrap is set to wrap, thsi determines how the next line is put together when top line runs out of space.
